# VM 115 and 120 Fix Summary (2025-12-07)

## Context
Followed the remediation pattern from `vm-114-fix-summary.md` while honoring the updated requirement to use DHCP (no static IPs) for VMs 114/115/120. VMs 108 and 116 were used as references only and not modified.

## VM 115 (k3s-server-2)
- Status before: EFI and storage mapped correctly to LVM; networking not configured inside guest (no IPv4 reported by agent).
- Changes applied:
  - `qm set 115 -ipconfig0 ip=dhcp`
  - `qm cloudinit update 115`

- Validation:
  - `qm agent 115 network-get-interfaces` shows DHCP IPv4: `192.168.4.34` on `eth0`.

  - `qm cloudinit update 120`
  - `qm start 120`
- Validation:
